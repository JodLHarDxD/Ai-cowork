## Hard Rules (apply to every agent, no exceptions)

### Tokens
- NEVER hardcode colors, spacing, durations, or font names — import from `@jodl/tokens`
- NEVER use inline styles — CSS modules or CSS custom properties only
- Token import pattern: `import { duration, ease } from "@jodl/tokens/motion"`

### Lenis / scroll
- NEVER use `overflow-x: hidden` on `body` or `html` — it breaks Lenis smooth scroll
- Use `overflow-x: clip` instead. This is a confirmed bug (see Codex Anti-Pattern #8 below)

### Reduced motion
- ALWAYS guard GSAP animations with `prefersReducedMotion` from `@jodl/tokens/motion`
- Set duration to `0` when `prefersReducedMotion` is true — never skip the guard

### React
- `inert` attribute: use `inert: true`, NOT `inert: ""`. React 18 treats `""` as falsy.
  Requires type augment: `interface HTMLAttributes<T> { inert?: boolean | undefined; }`
- Route-param state: ALWAYS reset derived UI state in `useEffect([param])` when route changes
- Context callbacks: ALWAYS wrap in `useCallback` to prevent consumer re-renders

## Spawn JSON Contract

**For orchestrators only.** Every orchestrator task MUST end with a ```json block containing this schema.
Missing or malformed JSON = runner cannot spawn children = build stalls.

```json
{
  "phase": "design | build | ship",
  "spawn-tasks": [
    {
      "role": "exact-role-slug",
      "brief": "Detailed task brief. Include: what to build, quality target, reference sites, constraints.",
      "depends-on": [],
      "parallel-group": 1
    }
  ],
  "ceo-report": "One paragraph summary for the CEO (human). Terse.",
  "ceo-decisions-needed": []
}
```

**Required fields on each task:** `role` (exact slug from routing-matrix.yaml) + `brief` (non-empty string).
**Parallel group:** tasks with same number run in parallel. Different numbers = sequential.
**Leaf agents** (frontend-master, backend-master, etc.) do NOT output spawn JSON — they output code/specs.

## Codex Anti-Patterns (Quick Checklist)

These 11 bugs were found in production code generated by Codex. Every implementation agent must check output against this list before finalizing.

1. **CSS token cascade** — `:root` tokens must default to light values; page-level classes override for dark.
   Red flag: components look fine on dark pages but are invisible on light pages.

2. **`inert` boolean** — use `inert: true` NOT `inert: ""`. Empty string is falsy in React 18.
   Red flag: keyboard can reach "closed" menu items, screen readers announce hidden content.

3. **Duplicate links** — card components with image + title both linking same URL = two identical tab stops.
   Fix: image gets `aria-hidden tabIndex={-1}`, only the named link is focusable.

4. **Missing state reset on route change** — `useEffect(() => { resetState() }, [slug])` required.
   Red flag: navigating product A → B shows A's selected size/color/image.

5. **Null guards on variant selection** — `size || product.sizes[0]` — fallback to first option.
   Red flag: cart gets `""` as variant value.

6. **JS state for CSS hover** — use `:hover` + `:focus-within` in CSS, not `useState` + onMouseEnter.
   Red flag: keyboard users can't trigger hover state.

7. **GSAP ScrollTrigger scoped kill** — `return () => tl.kill()` not `ScrollTrigger.getAll().kill()`.
   Red flag: cleanup destroys scroll triggers in unrelated components.

8. **`overflow-x: hidden` breaks Lenis** — use `overflow-x: clip` on `html` and `body`.
   Red flag: smooth scroll stops working, Lenis tracks wrong element.

9. **Unstable context callbacks** — wrap `openCart`, `closeCart` etc. in `useCallback`.
   Red flag: every cart state change re-renders all consumers.

10. **Missing `aria-label` on icon buttons** — all `+`, `-`, `✕`, icon-only buttons need `aria-label`.
    Red flag: screen readers read "plus" or "minus" with no context.

11. **Raw slug display** — capitalize or map slugs to display labels before rendering to users.
    Red flag: users see "women" instead of "Women" in breadcrumbs/categories.
